# カスタムメモリアロケータ

`malloc` / `free` 相当の機能を C23 で実装したサンプルです。

## 学べること

| テーマ | 内容 |
|---|---|
| ヒープ管理 | 固定サイズのメモリプールをブロックリストで管理する |
| フラグメンテーション | 空きブロックの断片化と **コアレッシング**（隣接ブロックの結合）で解消する仕組み |
| ポインタ演算 | ヘッダアドレス ± `sizeof(BlockHeader)` でユーザー領域を計算する |
| アロケーション戦略 | **ファーストフィット** と **ベストフィット** の比較 |
| デバッグ | `allocator_check_leaks()` による未解放ブロックの検出 |

## ファイル構成

```
custom_allocator/
├── allocator.h   — ブロックヘッダ構造体と公開 API の宣言
├── allocator.c   — ファーストフィット・ベストフィット・コアレッシングの実装
├── main.c        — 動作確認用のデモプログラム
└── Makefile      — ビルドとデモ実行
```

## ビルドと実行

```bash
cd custom_allocator
make run
```

`make clean` でビルド成果物を削除できます。

## 仕組みの解説

### メモリレイアウト

```
pool (静的配列 1 MiB)
┌──────────────┬──────────────────┬──────────────┬─────┐
│ BlockHeader  │  user data ...   │ BlockHeader  │ ... │
│  size        │                  │  size        │     │
│  is_free     │                  │  is_free     │     │
│  *next  ─────────────────────────►  *next      │     │
└──────────────┴──────────────────┴──────────────┴─────┘
```

各ブロックは **ヘッダ**（`BlockHeader` 構造体）と**ユーザーデータ領域**で構成されます。  
`my_malloc()` / `my_malloc_best_fit()` はヘッダの直後のアドレスをユーザーに返します。  
`my_free()` は受け取ったポインタを `(BlockHeader *)ptr - 1` で逆算してヘッダを取得します。

### ファーストフィット vs ベストフィット

| 方式 | 探索方法 | 速度 | フラグメンテーション |
|---|---|---|---|
| ファーストフィット | リスト先頭から最初に見つかった適合ブロックを使う | 速い | やや多い |
| ベストフィット | リスト全体を探索し、最小の適合ブロックを使う | 遅い | 少ない |

### コアレッシング（隣接ブロックの結合）

`my_free()` の後に `coalesce()` を呼び、**隣接する空きブロックを一つに結合**します。  
これにより、細かい空きブロックが積み重なる「外部フラグメンテーション」を抑えられます。

### 分割（スプリット）

要求サイズより大きなブロックを割り当てる際は `split_block()` で分割します。  
余った領域を新しい空きブロックとしてリストに追加することで、無駄なメモリを防ぎます。

### メモリリーク検出

```c
size_t leaks = allocator_check_leaks();
```

プールのブロックリストを走査し、`is_free == false` のブロックを標準エラー出力に表示します。  
戻り値が 0 ならリークなし、1 以上なら未解放ブロックの数です。

## C++ との比較

| C | C++ |
|---|---|
| `malloc` / `free`（手動） | `new` / `delete`（手動）または RAII（自動） |
| 型情報なし（`void *`） | 型付き（テンプレート） |
| カスタムアロケータは関数ポインタで渡す | `std::allocator` や Allocator コンセプト |
